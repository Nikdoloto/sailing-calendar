# Дизайн-система MVP — Календарь тренировок команды

Источник: приложенный пользователем визуальный файл дизайн-системы.

Этот документ является обязательной дизайн-спецификацией для разработки MVP. Интерфейс приложения должен использовать перечисленные ниже цвета, типографику, токены, компоненты и визуальные правила без вольной переработки.

## 1. Бренд и принципы

Название продукта: **Sail Team Calendar**

Смысл продукта: простой способ увидеть, когда команда свободна, и забронировать тренировку без поиска в Telegram.

Ключевые принципы интерфейса:

- **Просто** — минимум действий до отметки свободного слота.
- **Мобильно** — основной сценарий рассчитан на смартфон.
- **Видно всем** — доступность участников отображается общим командным состоянием.
- **Без лишней сложности** — без регистрации, сложных ролей и тяжёлых интеграций в MVP.

## 2. Цветовая система

| Роль | Название | HEX | Использование |
| --- | --- | --- | --- |
| Основной цвет | Глубокий морской синий | `#0D2B45` | Заголовки, бренд, основные акценты интерфейса |
| Акцент | Светлый голубой | `#4DA8E8` | Активные элементы, кнопки, PWA-блок, выбранные состояния |
| Успех / все свободны | Зелёный | `#22C55E` | Даты и слоты, где свободны все 4 участника |
| Частичное совпадение | Жёлтый | `#FACC15` | Даты и слоты, где свободны 3 из 4 участников |
| Нейтральный | Светло-серый | `#E6E9EE` | Даты и слоты, где свободны меньше 3 участников |
| Ошибка / недоступно | Мягкий красный | `#F87171` | Недоступность, ошибки, отрицательные статусы |

Правило подсветки календаря:

- Зелёный: есть слот, где свободны все 4 участника.
- Жёлтый: есть слот, где свободны 3 из 4 участников.
- Обычный/серый: свободны меньше 3 участников.

## 3. Типографика и UI-токены

### Типографика

| Токен | Размер / вес | Использование |
| --- | --- | --- |
| H1 | `24 / 700` | Главные заголовки экранов |
| H2 | `18 / 600` | Подзаголовки и названия разделов |
| Body | `16 / 400` | Основной текст интерфейса |
| Caption | `12 / 400` | Подписи, вспомогательный текст, метаданные |

### UI-токены

| Токен | Значение |
| --- | --- |
| Радиус карточек | `12px` |
| Базовые отступы | `4 / 8 / 12 / 16 / 24px` |
| Тень карточки | `0 4px 16px rgba(13, 43, 69, 0.08)` |

### Кнопки

- Primary: заливка акцентным голубым `#4DA8E8`, белый текст.
- Secondary: светлая/контурная кнопка с акцентным голубым состоянием.

### Чипы состояний

- `Свободны все 4` — зелёный.
- `3 из 4 свободны` — жёлтый.
- `Меньше 3` — нейтральный.
- `Недоступно` — красный.

## 4. Компоненты

### 4.1 Авторизация без регистрации

Сценарий:

1. Пользователь выбирает профиль.
2. Вводит PIN-код или пароль.
3. Нажимает кнопку `Войти`.
4. Вход запоминается на 30-90 дней.

Профили первой версии:

- Никита
- Даня
- Лиза
- Настя

Регистрация пользователей в MVP не нужна.

### 4.2 PWA-компонент

Компонент должен объяснять установку приложения на главный экран смартфона:

- Заголовок: `Добавить на главный экран`
- Подпись: `Установите приложение как ярлык на телефон.`

PWA должна позволять пользоваться приложением как отдельной иконкой на смартфоне.

### 4.3 Календарь месяца

Календарь должен быть оптимизирован под мобильный экран.

Обязательные особенности:

- Отдельно выделены пятница, суббота и воскресенье.
- Даты подсвечиваются по командной доступности.
- Переключение месяцев стрелками.
- Пользователь должен быстро видеть ближайшие подходящие даты.

### 4.4 Слоты дня

Базовые слоты:

| Слот | Время |
| --- | --- |
| Утро | `08:00-12:00` |
| День | `12:00-16:00` |
| Вечер | `16:00-20:00` |
| Весь день | `08:00-20:00` |

`Весь день` работает как быстрый выбор всех основных слотов дня.

### 4.5 Видимость участников в слоте

В каждом слоте должны быть видны участники, которые отметили доступность.

Формат отображения:

- имя участника;
- компактный аватар/иконка;
- цветовой индикатор доступности.

### 4.6 Статус тренировки

Если тренировка забронирована, показывается отдельный статусный блок:

- статус: `Тренировка забронирована`;
- дата;
- слот или точное время;
- инструктор;
- комментарий;
- признак, что редактирование доступно только Никите.

## 5. Экраны мобильного приложения

### 5.1 Экран входа

Содержит:

- логотип и название `Sail Team Calendar`;
- выбор пользователя;
- поле PIN-кода или пароля;
- кнопку `Войти`;
- подпись о запоминании входа на 30-90 дней.

### 5.2 Главный экран: календарь месяца

Содержит:

- заголовок месяца;
- календарную сетку;
- визуальное выделение пятницы, субботы, воскресенья;
- подсветку дат по доступности;
- подсказку-легенду:
  - зелёный: все 4 свободны;
  - жёлтый: 3 из 4 свободны;
  - серый: меньше 3 свободны.

### 5.3 Экран дня: слоты и видимость

Содержит:

- дату и день недели;
- слоты `Утро`, `День`, `Вечер`, `Весь день`;
- список доступных участников в каждом слоте;
- возможность текущему пользователю отметить или снять свою доступность.

### 5.4 Экран бронирования тренировки

Доступен только Никите.

Содержит:

- дату;
- слот или время;
- инструктора;
- необязательный комментарий;
- кнопку `Забронировать тренировку`;
- сообщение, что тренировка будет видна всем участникам.

## 6. MVP-границы

В первой версии не делаем:

- регистрацию;
- сложные роли;
- интеграцию с Google Calendar;
- уведомления;
- сложные права доступа;
- личные расширенные профили.

## 7. Технический контур MVP

Рекомендуемый контур:

- Frontend: веб-приложение + PWA.
- Backend: Node.js.
- Хранилище: SQLite.
- Web-сервер: Nginx.
- Авторизация: хэши паролей + cookie-сессия.

Nginx должен:

- отдавать frontend;
- проксировать API на Node.js backend.

Авторизация:

- без регистрации;
- вход по одному из четырёх заранее созданных профилей;
- сессия хранится в cookie;
- вход запоминается на 30-90 дней.
