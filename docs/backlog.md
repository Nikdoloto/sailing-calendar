| --- | --- | --- | --- |

## Developer Task: MVP-01..06 Backend

Контекст:

- Source of truth: `docs/spec.md`, `docs/technical-plan.md`, `docs/design-tokens.json`.
- Цель: реализовать backend API, SQLite-хранилище и тесты.
- Файлы: `server/**`, `tests/**`, `package.json`.

Критерии приемки:

- [ ] `npm start` запускает сервер.
- [ ] `npm test` проходит.
- [ ] Все API из технического плана реализованы.
- [ ] Пароли хранятся как хэши.
- [ ] Бронирование доступно только Никите.

## Developer Task: MVP-07..11 Frontend

Контекст:

- Source of truth: `docs/spec.md`, `docs/design-system.md`, `docs/design-tokens.json`.
- Цель: реализовать мобильный PWA-интерфейс.
- Файлы: `public/**`.

Критерии приемки:

- [ ] Экран входа показывает 4 профиля.
- [ ] Календарь месяца оптимизирован под мобильный вид.
- [ ] Weekend-дни визуально выделены.
- [ ] Доступность и бронирования обновляются через API.
- [ ] Есть manifest и service worker.

## Tester Task: MVP Acceptance

Проверить:

- вход валидным и невалидным PIN;
- отметку утро/день/вечер/весь день;
- зелёную подсветку 4 из 4;
- жёлтую подсветку 3 из 4;
- бронирование Никитой;
- запрет бронирования для остальных;
- PWA-файлы доступны как статические ресурсы.
